    GREEN phase — implement src/lib/csv/parse-listings.ts:

    Use PapaParse in string mode with `{ header: true, skipEmptyLines: true, transformHeader: (h) => h.trim().toLowerCase() }`.

    Define REQUIRED_COLS array: ['property_id', 'address', 'area', 'price_aed', 'bedrooms', 'bathrooms', 'size_sqft', 'status', 'developer', 'community', 'property_type'].

    Define ListingRowSchema with Zod:
    - property_id: z.string().min(1)
    - address: z.string().min(1)
    - area: z.string().min(1)
    - price_aed: z.coerce.number().positive()
    - bedrooms: z.coerce.number().nonnegative().int()
    - bathrooms: z.coerce.number().nonnegative().int()
    - size_sqft: z.coerce.number().positive()
    - status: z.enum(['available', 'sold', 'reserved'])
    - developer: z.string().min(1)
    - community: z.string().min(1)
    - property_type: z.string().min(1)
    - sold_date: z.string().optional()

    Export: `export type ListingRow = z.infer<typeof ListingRowSchema>`

    parseCsv(csvString: string): CsvValidationResult:
    1. Parse with PapaParse
    2. Check result.meta.fields for missing required columns — collect all missing cols, return early with errors if any missing
    3. Loop through result.data, safeParse each row with Zod, collect valid rows and error messages
    4. Error format: `Row ${i + 2}: ${error.issues.map(e => e.message).join(', ')}`
    5. Return { rows, errors }

    Export: `export { ListingRowSchema }` for reuse in route handler.
    Export: `export function parseCsv(csvString: string): CsvValidationResult`
  </implementation>

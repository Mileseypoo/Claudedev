Include in order:
1. Enable pgvector: `create extension if not exists vector with schema extensions;`
2. `uploads` table: id, tenant_id, filename, file_type (check: 'csv'|'pdf'), storage_path (nullable), status (check: 'processing'|'indexed'|'error'), error_message (nullable), row_count (nullable), chunk_count (nullable), created_at, updated_at. Index: (tenant_id, status).
3. `listings` table: id, tenant_id, property_id (text), address, area, price_aed (numeric(14,2)), bedrooms (integer), bathrooms (integer), size_sqft (numeric(10,2)), status (check: 'available'|'sold'|'reserved'), developer, community, property_type, sold_date (date nullable), created_at, updated_at. 4. `document_chunks` table: id, tenant_id, upload_id (references uploads(id) on delete cascade), chunk_index, content (text), embedding (extensions.vector(1536)), created_at. Index on tenant_id. HNSW index: `using hnsw (embedding extensions.vector_cosine_ops)`.
5. `listing_stats` table: tenant_id (primary key references tenants(id)), stats (jsonb default '{}'), updated_at.
6. Enable RLS on all 4 tables, create tenant_isolation policy on each.
7. `match_document_chunks` Postgres function (for Phase 3 use): accepts query_embedding vector(1536), match_threshold float, match_count int, p_tenant_id uuid. Returns table(id uuid, document_id uuid, content text, similarity float). Uses cosine similarity: `1 - (dc.embedding <=> query_embedding)`. Filters by tenant and threshold, orders by distance ASC, limits by match_count.
8. `recalculate_listing_stats(p_tenant_id uuid)` Postgres function: upserts into listing_stats with JSONB containing count_by_status, avg_price_by_bedrooms, price_range_by_area (min/max/median using percentile_cont(0.5)), recently_sold_count (status='sold' AND sold_date >= current_date - 30 days), total_listings.

Add header comment: "Run against Supabase SQL editor. v2 upgrade path: replace current_setting with auth.jwt()->>''tenant_id'' for JWT-based multi-tenant auth."

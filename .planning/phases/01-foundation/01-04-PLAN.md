    Create `src/hooks/useDeepgramStream.ts` implementing `useDeepgramStream`:
    - State: `{ connectionState: 'idle' | 'connecting' | 'connected' | 'disconnected', wsRef: WebSocket | null, chunkSequenceRef: number }`
    - `startStream(sessionId: string, mimeType: string)`:
      1. Fetch POST /api/deepgram-token to get { token, wsUrl }
      2. Build URL: append query params per Pattern 2 in research (`model=nova-2`, `smart_format=true`, `interim_results=true`; if mimeType includes 'opus': `encoding=opus&sample_rate=16000`)
      3. Open `new WebSocket(wsUrl, ['token', token])`; set `ws.binaryType = 'arraybuffer'`
      4. On `ws.onmessage`: parse JSON, if `result.type === 'Results' && result.is_final && transcript.trim()` → POST /api/session/chunk with `{ sessionId, sequence: chunkSequenceRef.current++, text: transcript, startSeconds: result.start }`
      5. On `ws.onopen`: set connectionState to 'connected'
      6. On `ws.onclose` / `ws.onerror`: set connectionState to 'disconnected'
    - `stopStream()`: send `JSON.stringify({ type: 'CloseStream' })` before closing; close ws
    - `pauseStream()`: send `JSON.stringify({ type: 'KeepAlive' })` per Deepgram docs to keep connection alive during MediaRecorder.pause()
    - `sendAudioChunk(blob: Blob)`: if ws.readyState === WebSocket.OPEN, convert Blob to ArrayBuffer and send
    - Heartbeat guard: if `ondataavailable` hasn't triggered in > 5000ms during an active session, assume mic is paused — call the `onMicInterrupted` callback (passed as option to `startStream`). This is the research-recommended supplement to visibilitychange.

    ANTI-PATTERN GUARD: Do NOT implement a WebSocket relay (browser → Next.js API → Deepgram). The browser connects directly to Deepgram. The Next.js API route only creates the short-lived token.

  <how-to-verify>
    **Prerequisites:**
    - Set the following environment variables in Vercel (or your hosting dashboard):
      - `NEXT_PUBLIC_SUPABASE_URL` — your Supabase project URL
      - `NEXT_PUBLIC_SUPABASE_ANON_KEY` — Supabase anon key
      - `SUPABASE_SERVICE_ROLE_KEY` — Supabase service role key
      - `DEEPGRAM_API_KEY` — Deepgram API key
      - `DEEPGRAM_PROJECT_ID` — Deepgram project ID
      - `DEFAULT_TENANT_ID` — the UUID of the POC tenant row you inserted into `public.tenants`
    - Run the DB migration: paste `db/migrations/001_foundation.sql` into the Supabase SQL editor and execute. Then insert the POC tenant row (see the commented seed block at the bottom of the migration file).

    **Test checklist (run on a physical iPhone or Android phone):**

    1. **PWA Install (PLAT-01)**
       - Open the HTTPS preview URL in Safari (iOS) or Chrome (Android)
       - iOS: tap Share → "Add to Home Screen" → confirm
       - Android: tap browser menu → "Install app" or "Add to Home Screen"
       - Confirm: app opens in standalone mode (no browser chrome), dark background visible

    2. **Microphone permission (SESS-04)**
       - Tap "Start Session" on the home screen
       - Confirm: consent modal appears with "Client has consented to recording" text
       - Tap "Start Recording"
       - Confirm: OS microphone permission prompt appears
       - Grant permission
       - Confirm: app navigates to active session screen

    3. **Live transcript (SESS-01, PLAT-01)**
       - Speak a sentence (e.g., "How many properties are available in Dubai Marina?")
       - Confirm: transcript text appears within 3 seconds of speaking
       - Confirm: pulsing mic indicator is visible with animation

    4. **Recording indicator (SESS-03)**
       - During active recording: confirm pulsing mic indicator is clearly visible
       - Confirm: session timer is counting up

    5. **Pause and Resume (SESS-05)**
       - Tap "Pause" (left button)
       - Confirm: recording pauses (Deepgram KeepAlive sent), pause is reflected in UI
       - Speak a few words — confirm no new transcript appears while paused
       - Tap "Resume"
       - Confirm: recording resumes, transcript continues

    6. **Screen lock / WakeLock (SESS-03, PLAT-01)**
       - During active recording, lock the phone screen
       - iOS 18.4+: confirm screen stays on (WakeLock active)
       - iOS < 18.4 or any iOS: confirm "Keep your screen on" banner is visible if screen locks
       - Unlock the phone and return to the app
       - Confirm: "Recording stopped" banner with Resume button appears (no auto-resume)
       - Tap Resume and confirm recording continues

    7. **Session recovery (SESS-06)**
       - During active recording, force-close the app (swipe up to kill on iOS)
       - Re-open the app
       - Confirm: recovery screen appears ("Session in progress" / "Resume session" / "Start fresh")
       - Tap "Resume session"
       - Confirm: app navigates to active session screen with the same session (transcript intact in Supabase)

    8. **End session (SESS-02)**
       - Tap "End" (right button)
       - Confirm: End button label changes to "Confirm end" (secondary tap required)
       - Tap "Confirm end"
       - Confirm: app returns to home screen, session marked as ended in Supabase

    **Multi-tenancy smoke test (PLAT-02):**
    - In Supabase Table Editor: check `sessions` table — confirm all rows have the correct `tenant_id` (the POC tenant UUID)
    - Confirm no row exists with a null `tenant_id`

    **Note any issues:** If a step fails, describe what happened and which device/OS version you were using.
